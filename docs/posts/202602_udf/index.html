STANPUMP again

In my previous blog post, I took a deep dive into STANPUMP’s source code to understand the underlying algorithms for Target-Controlled Infusion (TCI) and reimplemented them in Python. However, we only glimpsed over the actual core algorithm which calculates the required infusion rates to achieve target concentrations.

This post will show how the TCI algorithm works step by step, including:

  • Initial bolus dosing
  • Target maintenance
  • Target increases
  • Target decreases

The goal is to understand every step of the algorithm that runs every 10 seconds in operating rooms around the world.

The Unit Disposition Function (UDF)

The UDF is the key to TCI’s efficiency and elegance. It answers a simple question:

What happens if I infuse 1 μg/s for 10 seconds, then stop?

The UDF is pre-calculated once at the start of the case using these steps:

  1. Initialize: Start with empty compartments (all zeros)
  2. Simulate a unit rate infusion: Run model with rate = 1 μg/s for delta_seconds (typically 10s)
  3. Stop and observe: Set rate = 0 and continue simulation
  4. Find peak: Record when effect site concentration reaches its maximum
  5. Store trajectory: Save the concentration at each time point up to the peak

For the Gepts Sufentanil model, this calculation shows:

  • Peak occurs at 428 seconds
  • The peak concentration is approximately 0.19 μg/L
  • The shape reflects the complex interplay of distribution and elimination

The Power of Linearity

Because the pharmacokinetic system is linear, the UDF enables elegant scaling:

Want to achieve 0.3 μg/L at the peak?

  • UDF shows 1 μg/s -> 0.19 μg/L at peak
  • Therefore you need 0.3 / 0.19 = 1.58 μg/s for 10 seconds
  • This amount to a bolus of 15.7 μg over the first 10 seconds
  • Formula: rate = target / udf[peak_time]

This is the foundation of the TCI algorithm: scale the unit response to achieve any target.

Single initial bolus dose

In the simple case, when only a bolus should be administered

  • The rate is calculated using rate = target / udf[peak_time]
  • Rate is maintained for initial 10 seconds
  • Rate is then set to zero (infusion stops)
  • Drug redistributes between compartments
  • Effect site concentration rises
  • At peak time, Ce converges to target
  • After peak time, Ce decreases as drug is distributed and metabolized

Implementing the TCI Algorithm

If you want to continue infusion after the initial bolus, an iterative algorithm is needed where the infusion rate is recalculated at every time step. Let’s go through the complete TCI algorithm, explaining each step.

Define the Pharmacokinetic Model

We will use the Gepts Sufentanil model again. This code uses effect site concentrations but the same algorithm applies to plasma concentration targets.

import numpy as np
import tci

# Gepts Sufentanil: 3-compartment model with effect site
gepts = tci.Parameters(
    k10=0.0645,  # min⁻¹ - Elimination from central compartment
    k12=0.1086,  # min⁻¹ - Transfer: central → peripheral 1
    k13=0.0229,  # min⁻¹ - Transfer: central → peripheral 2 (slow)
    k21=0.0245,  # min⁻¹ - Transfer: peripheral 1 → central
    k31=0.0013,  # min⁻¹ - Transfer: peripheral 2 → central (very slow!)
    ke0=0.112,   # min⁻¹ - Effect site equilibration
    vc=14.3,     # L - Central compartment volume
)

Convert Time Units and Pre-compute System Properties

# Convert to seconds for 10-second update intervals
model_params = gepts.per_seconds()
delta_seconds = 10

# Calculate eigenvalues (hybrid rate constants)
lambdas = tci.calculate_lambdas(model_params)

# Pre-calculate UDFs and coefficients
udfs, coefs, peak_time = tci.calculate_udfs(model_params, lambdas, delta_seconds)
assert peak_time == 428  # Gepts-specific peak time

udf = np.array(udfs[1])  # Extract effect site UDF

The calculate_lambdas() function constructs the system matrix and computes its eigenvalues. Charles Minto recently wrote a comprehensive document which explains exactly how the eigenvalues are calculated in the STANPUMP code.

The Main Simulation Loop

The TCI algorithm runs every delta_seconds (10 seconds), recalculating the infusion rate based on the current target. In this simulation, I set an intial target at 0.2 ug/L, after 500 seconds it is increased to 0.3 and after 1000 seconds it is lowered to 0.1.

# Initialize simulation state
Ce = np.zeros(4)
time = [0]
targets = [0]
rates = [0]
analytic_ce = [0]

simulation_time = 4000  # 4000 seconds total

for i in range(simulation_time // delta_seconds):
    t = i * delta_seconds
    
    # ------------------------------------------------------------------------
    # Clinician sets target (simulating clinical decision-making)
    # ------------------------------------------------------------------------
    if t <= 500:
        current_target = 0.2  # Initial induction
    elif t <= 1000:
        current_target = 0.3  # Increase
    else:
        current_target = 0.1  # Decrease
    
    targets.append(current_target)
    
    # ------------------------------------------------------------------------
    # Calculate required infusion rate
    # ------------------------------------------------------------------------
    rate = calculate_rate(Ce, current_target, lambdas, udf, coefs, 
                          peak_time, delta_seconds)
    rates.append(rate)
    
    # ------------------------------------------------------------------------
    # Apply infusion and update compartments
    # ------------------------------------------------------------------------
    Ce = tci.update(Ce, lambdas, coefs.e_coef, rate, delta_seconds)
    time.append(t + delta_seconds)
    analytic_ce.append(np.sum(Ce))

The Iterative Rate Calculation Algorithm

This is where the magic happens. The goal is to find an infusion rate such that the peak effect site concentration equals the target.

Why Iteration is Necessary

  • Peak time is dynamic: It depends on current compartment state AND infusion rate
  • From empty state → peak at 428s
  • From loaded state → peak might be at 60s, 120s, 200s…
  • We must find the (rate, peak_time) pair that satisfies our goal

The Algorithm

def calculate_rate(Ce, current_target, lambdas, udf, coefs, 
                   peak_time, delta_seconds):
    """
    Calculate infusion rate to achieve target concentration at peak.
    Uses iterative algorithm to find convergence.
    """
    
    ok_condition = False
    peak_time_tmp = peak_time  # Start with UDF peak as initial guess
    rate_candidate = 0
    
    while not ok_condition:
        # ----------------------------------------------------------------
        # Step 1: Predict concentration at estimated peak (no infusion)
        # ----------------------------------------------------------------
        # "Where will we be if we don't give any drug?"
        virtual = virtual_model(Ce, peak_time_tmp, lambdas)
        
        # ----------------------------------------------------------------
        # Step 2: Calculate the gap
        # ----------------------------------------------------------------
        residual = current_target - virtual
        
        # ----------------------------------------------------------------
        # Step 3: Calculate candidate rate using linearity
        # ----------------------------------------------------------------
        rate_candidate = residual / udf[peak_time_tmp]
        
        # ----------------------------------------------------------------
        # Step 4: Handle target decrease (early exit)
        # ----------------------------------------------------------------
        # Cannot "un-give" drug - set rate to 0 and let elimination work
        if rate_candidate <= 0:
            rate_candidate = 0
            ok_condition = True
            continue
        
        # ----------------------------------------------------------------
        # Step 5: Find actual peak time with this rate
        # ----------------------------------------------------------------
        # Peak shifts based on current state + proposed rate
        peak_time_tmp = find_peak(
            peak_time_tmp, rate_candidate, Ce, peak_time,
            lambdas, udf, delta_seconds
        )
        
        # ----------------------------------------------------------------
        # Step 6: Recalculate at NEW peak time
        # ----------------------------------------------------------------
        virtual = virtual_model(Ce, peak_time_tmp, lambdas)
        predicted_ce = virtual + udf[peak_time_tmp] * rate_candidate
        
        # ----------------------------------------------------------------
        # Step 7: Check convergence (STANPUMP criterion)
        # ----------------------------------------------------------------
        # Tolerance: 0.00001% of target (essentially machine precision)
        ok_condition = np.abs(predicted_ce - current_target) <= 1e-7 * current_target
        
        # If not converged: loop repeats with updated peak_time_tmp
        # Typically converges in 2-5 iterations
    
    return rate_candidate

Supporting Functions

Virtual Model: Projects current state forward in time

def virtual_model(A, t, lambdas):
    """
    Predict drug concentration at time t given current state.
    Assumes no infusion (rate = 0).
    """
    decay = np.exp(-lambdas * t)
    return np.sum(A * decay)

Find Peak: Searches for concentration maximum

def find_peak(current_time, rate, Ce, peak_time, lambdas, udf, delta_seconds):
    """
    Find when effect site concentration peaks with given rate.
    Uses hill-climbing search.
    """
    # Calculate concentrations at three time points
    current = (virtual_model(Ce, current_time, lambdas) + 
              udf[current_time] * rate)
    earlier = (virtual_model(Ce, current_time - 1, lambdas) + 
              udf[current_time - 1] * rate)
    later = (virtual_model(Ce, current_time + 1, lambdas) + 
            udf[current_time + 1] * rate)
    
    # Search for local maximum
    while current < earlier or current < later:
        if current < earlier:
            if current_time == delta_seconds:
                return current_time
            current_time -= 1
            later = current
            current = earlier
            earlier = (virtual_model(Ce, current_time - 1, lambdas) + 
                      udf[current_time - 1] * rate)
        else:
            current_time += 1
            earlier = current
            current = later
            later = (virtual_model(Ce, current_time + 1, lambdas) + 
                    udf[current_time + 1] * rate)
    
    return current_time

Step 5: Compartment Update

After calculating the rate, we apply the infusion for delta_seconds:

def update(A, lambdas, coefs, rate, dt):
    """
    Update compartment amounts after infusion at given rate for time dt.
    
    This is the discrete-time solution:
    A(t+dt) = A(t)·exp(-λ·dt) + c·rate·(1 - exp(-λ·dt))
    """
    decay = np.exp(-lambdas * dt)
    return A * decay + coefs * rate * (1 - decay)

This formula is the closed form solution of the PKPD differential equations.

How Different Scenarios Are Handled

Scenario 1: Initial Bolus (From Zero State)

Situation: All compartments empty, target = 0.2 μg/L

Algorithm behavior: 1. virtual = 0 (starting from zero) 2. residual = 0.2 - 0 = 0.2 3. rate = 0.2 / udf[428] ≈ 0.4 μg/s 4. Peak occurs at 428s (unchanged, as predicted) 5. Infuse 0.4 μg/s for 10s, then rate = 0 6. Concentration rises over next 7 minutes to ≈0.2 μg/L

Scenario 2: Target Increase (0.2 → 0.3 μg/L)

Situation: Compartments partially loaded, need higher target

Algorithm behavior: 1. Iteration 1: - Extrapolate to peak_time = 428s with rate = 0 - Predict concentration ≈ 0.19 μg/L (slight decay) - residual = 0.3 - 0.19 = 0.11 - rate = 0.11 / udf[428] - Find new peak → say 180s (much earlier!)

  1. Iteration 2:
    • Extrapolate to 180s with rate = 0
    • Different predicted concentration
    • Recalculate rate
    • Find new peak → say 165s
  2. Convergence (typically 2-5 iterations):
    • Rate settles at value that produces exactly 0.3 μg/L at the (now known) peak time
    • Effect site concentration rises to new target

Scenario 3: Target Maintenance

Situation: At target, need to compensate for elimination

Algorithm behavior: 1. Small residual (elimination creates deficit) 2. Calculate low maintenance rate 3. Peak time relatively stable 4. Quick convergence (already near target) 5. Small rate just balances elimination

Scenario 4: Target Decrease (0.3 → 0.1 μg/L)

Situation: Current concentration above new target

Algorithm behavior: 1. residual = 0.1 - 0.3 = -0.2 (negative!) 2. rate_candidate = -0.2 / udf[peak] < 0 3. Early exit: rate = 0 (cannot remove drug) 4. No iteration needed 5. Natural elimination reduces concentration 6. When concentration falls below 0.1, algorithm resumes with positive rates

Computational Efficiency

Notice the computational strategy:

Done once (pre-computation): - Calculate eigenvalues: O(n³) for n×n matrix - Generate UDF array: ~500 iterations - Calculate coefficients: Simple arithmetic

Done every 10 seconds (real-time): - Project state forward: Simple exponentials - Array lookups in UDF - 2-5 iterations to convergence - Each iteration: ~10 arithmetic operations

The expensive linear algebra happens once. The real-time algorithm is just arithmetic and array access—fast enough to run on a 1990s microcontroller.

Visualizing the Results

import matplotlib.pyplot as plt

fig, ax = plt.subplots()

ax.plot(time, targets, label="Target", linewidth=2, color='red', linestyle='--')

The plot reveals:

  1. Initial bolus: High rate spike to rapidly achieve 0.2 μg/L
  2. Plateau phase: Low maintenance rate compensates for elimination
  3. Target increase: Rate spike to reach 0.3 μg/L
  4. Target decrease: Rate drops to zero, concentration decays naturally
  5. Tracking accuracy: Achieved concentration closely follows target
